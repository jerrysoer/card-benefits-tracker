01 The Agentic Loop

Think. Act. Verify. Repeat.

Claude Code operates in a continuous cycle. It gathers context, takes action with tools, verifies the results, and iterates until the task is complete. You can interrupt and steer at any point.

1

You prompt

Describe what you want in natural language. Fix a bug, add a feature, refactor a module, explain some code. As specific or open-ended as you like.

"Fix the login timeout bug"
2

Claude reasons

Extended thinking activates. Claude breaks down the problem, plans its approach, and decides which tools to use. Reasoning scales from quick to deep analysis depending on complexity.

Extended Thinking Plan Decomposition
3

Tools execute

Claude calls the right tools — reading files, searching code, editing source, running commands, fetching documentation. Multiple tools can fire in parallel for speed.

Read Edit Bash Grep WebSearch
4

Verify results

Run tests, check outputs, review diffs. Claude confirms the changes work before moving on — or identifies what went wrong.

npm test Diff Review
5

Iterate or complete

If the task isn't done, loop back. Fix failing tests, refine the approach, handle edge cases. Continue until everything passes.

Loops until complete
02 Tool Belt

Built-in capabilities

Six categories of tools give Claude direct agency over your codebase and development workflow. Each tool is purpose-built for its task.

File Operations Read · Edit · Write
Read, edit, and write files directly. Surgical line-level edits, not full rewrites. Supports images, PDFs, and Jupyter notebooks.
Code Search Glob · Grep
Find files by pattern, search contents with regex. Powered by ripgrep for instant results across any codebase size.
Shell Execution Bash
Run any shell command — tests, builds, linters, scripts. Persistent working directory across calls. Optional OS-level sandboxing.
Web Access WebSearch · WebFetch
Search the web for current information and fetch documentation. Keeps Claude up-to-date beyond its training data cutoff.
Subagents Task · Explore · Plan
Spawn isolated workers with their own context window. Run research in background, parallelize tasks, protect the main conversation.
Git & GitHub git · gh CLI
Create commits, manage branches, open pull requests, review diffs. Full git workflow from staging to push, with safety guardrails.
03 Permission Model

You stay in control

A tiered system ensures Claude asks before taking risky actions. Reading is free. Edits and commands need your say-so.

Tier Tools Approval
Read-only Read, Grep, Glob, WebSearch Auto-allowed
File edits Edit, Write, NotebookEdit Ask once per session
Shell Bash execution Ask each time
Shift+Tab — Permission Modes
Default Normal. Ask before edits and commands.
Auto-Edit Auto-accept edits. Still asks for shell commands.
Plan Read-only analysis. No modifications allowed.
Bypass Full autonomy. Skip all prompts. Sandboxed use only.

04 Context Management

Memory that persists

Five layers of context ensure Claude always knows your project conventions, team rules, and personal preferences — across every session.

CLAUDE.md
Loaded every session. Define coding standards, architecture rules, and team conventions. Supports file imports with @filename.
Modular Rules
The .claude/rules/ directory holds organised, path-specific instructions. Rules with frontmatter only load when matching files are touched.
Auto Memory
Claude maintains its own notes at ~/.claude/projects/. Patterns, debugging insights, and lessons learned persist across sessions automatically.
Compaction
As context approaches limits, older messages are automatically summarised. Conversations are never limited by the context window.
Layered Settings
Configuration cascades: Managed (org) → CLI flags → Project local → Project shared → User. Deny rules always win.
05 Workflow

From idea to shipped code

A typical session flows through four phases. Jump in at any point or skip phases entirely.

01

Plan

Enable Plan Mode for read-only exploration. Claude researches your codebase, proposes an approach, and you iterate before any code changes.

02

Implement

Switch to normal mode. Claude edits files, creates new ones, and runs builds. Multiple subagents can work in parallel on independent tasks.

03

Test

Claude runs your test suite, catches failures, and fixes them in a loop. Edge cases get handled. Linting and type-checking pass cleanly.

04

Ship

Create a well-crafted commit, open a pull request with a proper description, and push. Claude follows your git conventions automatically.
